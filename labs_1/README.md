Лаба по Хаффману
Сжал файлы без потерь по алгоритму Хаффмана, идея с Хабра (https://habr.com/ru/articles/144200/)

Сжимает файлы в .zhm с водяным знаком ZHM! (4 байта, чтобы знать, что мой).
Пишет логи в huffman.log (время, размеры, процент сжатия).
Показывает таблицу кодов (символ: код) в консоли и логе.
Проверяет, совпадает ли декодированный файл с оригиналом.
Добавляет паддинг (0–7 бит), чтобы всё было кратно байтам.
Ловит ошибки (битый .zhm, нет ZHM!).

Как юзать

Клони репо:git clone https://github.com/kb-nik/AECI_labs.git
cd AECI_labs/labs_1


Сжать:python3 main.py encode input.txt compressed


Распаковать:python3 main.py decode compressed.zhm output.txt



Тесты

Большой файл: 302505 байт → 100849 байт (66% сжатия).
Маленький: 15 байт → 19 байт (хуже из-за ZHM! и дерева).
Декодирование совпадает с оригиналом (проверял через diff).

Логи
Пример из huffman.log:
2025-10-15 03:12:34 - Code: 'b' -> 00
2025-10-15 03:12:34 - Code: 'e' -> 11
2025-10-15 03:12:34 - Encode: input.txt (15B) -> compressed.zhm (19B), -26.67% saved

Замечания

Маленькие файлы не сжимаются из-за оверхеда (ZHM!, дерево, паддинг).
Водяной знак ZHM! защищает от подмены.
